PROJECT INFORMATION

Title: Privacy and Security of IoT Technologies for Visually Impaired

RESEARCH QUESTION
═══════════════════════════════════════════════════════════════════════
How can we design a secure, privacy-preserving, and power-efficient 
communication protocol for IoT assistive devices serving visually 
impaired users?

METHODOLOGY
═══════════════════════════════════════════════════════════════════════
1. Expert Interviews (3 professionals):
   → Dr. Gulnara Abitova (Policy & Ethics)
   → Mahdi Habibi (IoT Development)
   → Altynbek Kabiyev (Network Security)

2. Problem Identification:
   → MQTT/CoAP run without encryption
   → BLE has weak pairing mechanisms
   → No privacy-by-design in existing protocols
   → Power vs. security trade-offs unresolved

3. Solution Design:
   → Created SATP protocol based on expert feedback
   → Built working C++ prototype
   → Validated through demonstration scenarios

KEY FINDINGS
═══════════════════════════════════════════════════════════════════════
Problem                          SATP Solution
─────────────────────────────────────────────────────────────────────
Optional encryption              → Mandatory encryption (all messages)
Weak authentication             → Certificate-based + mutual auth
Power consumption               → Adaptive security (4 levels)
No privacy focus                → Privacy-by-design architecture
Accessibility gaps              → Audio feedback, simple config
Safety concerns                 → Priority messaging, redundancy

PROTOCOL SPECIFICATIONS
═══════════════════════════════════════════════════════════════════════
Name:          SATP (Secure Assistive Transport Protocol)
Base:          UDP + DTLS (recommended by experts)
Security:      4 adaptive levels (Maximum → Emergency)
Overhead:      56 bytes per message
Latency:       <50ms target for critical messages
Battery Impact: <5% additional vs. unsecured UDP

TECHNICAL HIGHLIGHTS
═══════════════════════════════════════════════════════════════════════
→ Mandatory end-to-end encryption (addresses MQTT/CoAP vulnerability)
→ Power-aware security adaptation (4 levels based on battery)
→ Privacy-by-design (location anonymization, data minimization)
→ Safety-critical features (priority messaging, redundancy)
→ Real-time performance (<50ms latency for navigation)
→ Accessibility focused (audio feedback, simple setup)
